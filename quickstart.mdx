## Get Started in Three Steps

This guide will help you deploy Postiz, connect your first social platform, and schedule your first post.

<Steps>
  <Step title="Deploy Postiz">
    Choose your deployment method:

    <Tabs>
      <Tab title="Cloud (Fastest)">
        Sign up for a free account at [platform.postiz.com](https://platform.postiz.com) and skip to Step 2.
        
        <Info>
          The cloud version is immediately ready with no setup required. Perfect for testing Postiz.
        </Info>
      </Tab>
      
      <Tab title="Docker (Self-Hosted)">
        Deploy locally with Docker Compose:

        ```bash
        # Clone the repository
        git clone https://github.com/gitroomhq/postiz-app.git
        cd postiz-app

        # Start all services
        docker compose up -d
        ```

        Access Postiz at `http://localhost:4007`

        <Note>
          Full installation details are in our [Installation Guide](/installation).
        </Note>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Create Your Account">
    On first launch, you'll be prompted to create an admin account.

    **Required Information:**
    - Full name
    - Email address
    - Password (min 8 characters)

    <Warning>
      If `RESEND_API_KEY` is configured in your environment, you'll need to verify your email. Otherwise, accounts are auto-activated.
    </Warning>

    After creating your account, you'll land on the main dashboard.
  </Step>

  <Step title="Connect Your First Platform">
    Let's connect X (Twitter) as your first platform:

    1. Navigate to **Settings** → **Integrations**
    2. Click **Add Channel**
    3. Select **X (Twitter)** from the platform list
    4. Click **Connect with OAuth**

    <CodeGroup>
      ```bash .env (Self-hosted)
      # Add these to your .env file before connecting
      X_API_KEY="your_twitter_api_key"
      X_API_SECRET="your_twitter_api_secret"
      ```
    </CodeGroup>

    <Info>
      **Don't have X API credentials?** Follow [Twitter's Developer Portal](https://developer.twitter.com/en/portal/dashboard) to create an app and get your keys.
    </Info>

    5. Authorize Postiz to access your X account
    6. Your account will appear in the **Connected Channels** list

    <Tip>
      You can connect multiple accounts from the same platform (e.g., personal and business Twitter accounts).
    </Tip>
  </Step>

  <Step title="Schedule Your First Post">
    Now let's create and schedule your first post:

    1. Click the **+ New Post** button in the top navigation
    2. Select your connected X account from the channel selector
    3. Type your post content in the editor

    **Using AI Content Generation (Optional):**
    
    If you have `OPENAI_API_KEY` configured:
    
    - Click the **AI Assist** button
    - Enter a topic or prompt
    - Postiz will generate engaging content for you

    4. **Set Your Schedule:**
       - Click the calendar icon
       - Choose date and time
       - Select your timezone

    5. **Preview Your Post:**
       - Click **Preview** to see how it will appear
       - Make any adjustments

    6. **Schedule It:**
       - Click **Schedule Post**
       - Your post is now queued!

    <Success>
      Congratulations! Your first post is scheduled. Postiz will automatically publish it at the specified time.
    </Success>
  </Step>
</Steps>

## Platform-Specific Setup

Each platform requires its own OAuth credentials. Here are quick setup guides:

<AccordionGroup>
  <Accordion title="X (Twitter)" icon="x-twitter">
    **Required Credentials:**
    - API Key
    - API Secret

    **Setup Steps:**
    1. Go to [Twitter Developer Portal](https://developer.twitter.com/en/portal/dashboard)
    2. Create a new app
    3. Enable OAuth 2.0
    4. Set callback URL: `http://localhost:4007/integrations/social/x` (adjust for your domain)
    5. Copy credentials to your `.env` file

    ```bash
    X_API_KEY="your_api_key"
    X_API_SECRET="your_api_secret"
    ```
  </Accordion>

  <Accordion title="LinkedIn" icon="linkedin">
    **Required Credentials:**
    - Client ID
    - Client Secret

    **Setup Steps:**
    1. Visit [LinkedIn Developers](https://www.linkedin.com/developers/)
    2. Create a new app
    3. Add OAuth 2.0 redirect: `http://localhost:4007/integrations/social/linkedin`
    4. Request access to "Share on LinkedIn" product

    ```bash
    LINKEDIN_CLIENT_ID="your_client_id"
    LINKEDIN_CLIENT_SECRET="your_client_secret"
    ```
  </Accordion>

  <Accordion title="Instagram" icon="instagram">
    **Required Credentials:**
    - Facebook App ID
    - Facebook App Secret

    **Setup Steps:**
    1. Create a [Facebook App](https://developers.facebook.com/)
    2. Add Instagram Basic Display product
    3. Add Instagram Graph API
    4. Configure OAuth redirect to your Postiz instance

    ```bash
    FACEBOOK_APP_ID="your_app_id"
    FACEBOOK_APP_SECRET="your_app_secret"
    ```

    <Warning>
      Instagram requires a Facebook Business Page connected to your Instagram account.
    </Warning>
  </Accordion>

  <Accordion title="YouTube" icon="youtube">
    **Required Credentials:**
    - Google Client ID
    - Google Client Secret

    **Setup Steps:**
    1. Go to [Google Cloud Console](https://console.cloud.google.com/)
    2. Create a new project
    3. Enable YouTube Data API v3
    4. Create OAuth 2.0 credentials
    5. Add redirect: `http://localhost:4007/integrations/social/youtube`

    ```bash
    YOUTUBE_CLIENT_ID="your_client_id"
    YOUTUBE_CLIENT_SECRET="your_client_secret"
    ```
  </Accordion>

  <Accordion title="TikTok" icon="tiktok">
    **Required Credentials:**
    - Client Key
    - Client Secret

    **Setup Steps:**
    1. Register at [TikTok for Developers](https://developers.tiktok.com/)
    2. Create a new app
    3. Request video publishing permissions
    4. Configure redirect URI

    ```bash
    TIKTOK_CLIENT_ID="your_client_key"
    TIKTOK_CLIENT_SECRET="your_client_secret"
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="OAuth redirect errors">
    **Problem:** Getting a redirect URI mismatch error when connecting platforms.

    **Solution:** Ensure your callback URLs in platform developer consoles match exactly:
    
    ```bash
    # Your .env file
    FRONTEND_URL="http://localhost:4007"
    
    # Platform redirect should be:
    # http://localhost:4007/integrations/social/{platform_name}
    ```

    Replace `localhost:4007` with your actual domain in production.
  </Accordion>

  <Accordion title="Posts not publishing">
    **Problem:** Scheduled posts aren't being published at the specified time.

    **Solution:** Check that Temporal orchestrator is running:
    
    ```bash
    # Check service status
    docker compose ps
    
    # Should see 'temporal' service as 'running'
    # Check orchestrator logs
    docker compose logs orchestrator
    ```

    The orchestrator handles all background job execution.
  </Accordion>

  <Accordion title="Email verification not working">
    **Problem:** Not receiving verification emails after signup.

    **Solution:** If you don't have Resend configured, disable email verification:
    
    ```bash
    # Remove or comment out in .env
    # RESEND_API_KEY="..."
    ```

    Users will be auto-activated without email verification.
  </Accordion>
</AccordionGroup>
